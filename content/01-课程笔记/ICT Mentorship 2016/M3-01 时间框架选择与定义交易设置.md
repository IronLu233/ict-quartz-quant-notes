## 章节概要

- `00:00-04:40` 引入主题：第三个月第一课，讨论时间框架选择，以及如何为自己的模型定义交易设置
- `04:41-21:25` 时间框架分层：月线用于持仓，周线用于波段，日线用于短期交易的主分析框架，4H 及以下偏执行
- `20:16-21:25` 关键建议：即使未来想做日内，也建议先从日线建立市场偏见与方向框架
- `50:37-54:30` setup 收敛：最终应压缩成少数自己真正看得懂、能重复执行的几类入场模型

## 笔记

这节课开宗明义讲两件事：第一，交易者该如何选择适合自己的时间框架；第二，如何把零散观察收束成自己真正会交易的 setup。ICT 的重点并不是单纯告诉你“看哪张图”，而是在强调：高时间框架负责给出方向与前提，低时间框架负责把这个前提转化为可执行的入场。

### 1. 日线图与 4H 都要看，但分工不同

- 这节课里我更关心、也更值得记住的重点，就是 ICT 对 `日线图框架` 的强调，但这不代表只看日线就够了
- 如果目标是做 `日内交易`，那更合理的理解应该是：`日线` 负责给出大方向与结构前提，`4H` 负责把这个前提进一步细化到更接近执行的位置
- 日线足够高，能看到机构更在意的水平位、流动性、止损分布、FVG 与订单流方向
- 4H 则更适合观察日内交易前真正会用到的局部结构，例如价格是否已经接近日线给出的关键区域、是否出现更清楚的回补与位移
- 所以这里不是“日线和 4H 二选一”，而是它们分别承担不同层级的工作：日线定框架，4H 做过渡，低时间框架再负责触发

### 2. ICT 对时间框架的基本划分

- `月线` 更多用于长线持仓交易，适合不愿意承受低时间框架噪音的人
- `周线` 更适合波段交易，交易频率仍然较低，但比月线更容易参与
- `日线` 是短期交易的最佳分析起点，可以同时保留高时间框架的结构感与足够多的参考点
- `4小时` 对日内交易者来说很重要，它既承接日线框架，又比日线更接近实际执行
- `4小时以下` 更偏向日内触发与执行层，本课不会展开太多细节，但已经明确其角色不是替代高时间框架分析

### 3. 对我更有价值的，不是泛泛谈时间框架，而是下跌途中的海龟汤模式

- 这节课里，对我真正有价值的重点，不只是“先看哪张图”，而是 `下跌途中的海龟汤` 这种更具体的 setup
- 这个模式的吸引力在于，它不是逆着更大框架随意抄底，而是先承认市场总体处于下跌路径中，再去等待一次针对局部低点止损的清扫
- 也就是说，先有高时间框架的看跌前提，再去观察局部扫低、回收、回到区间内或回到关键区域后的反应
- 这样一来，海龟汤就不再只是一个孤立的“假突破形态”，而是下跌结构中的一次可利用流动性事件
- 如果后面继续细化，这种模式很适合和我之前反复关注的 `OTE + 止损模式` 结合起来理解
- 如果沿用上一节课延伸出来的术语，我觉得这类结构也可以称为 `弹簧`：价格先被压缩，接着扫掉一侧止损，随后快速反向释放
- 这样命名的好处是，它比“假突破”更强调事件顺序，也更适合后续写成规则：`压缩 -> 清扫 -> 回收 -> 释放`

![[M3-01_海龟汤扫止损.jpg]]

### 4. 54 分钟左右最关键的地方：ICT 说自己现在只交易三种模式

- 这段对我尤其重要，因为 ICT 明确说他现在真正交易的只有三种模式，除此之外基本不做别的；如果从量化角度看，这三种东西很可能就是比“图形名称”更底层的交易因子
- 第一种是 `订单块回撤`：当高时间框架给出方向后，价格回到先前暴露出来的看涨或看跌订单块，在那里顺着大方向介入
- 第二种是 `海龟汤 / 止损清扫`：如果价格冲过前高或跌破前低，触发一侧止损，然后迅速回收，那么这个扫损事件本身就可以成为入场依据
- 第三种是 `流动性真空回补后的再扩张`：价格先拉出一段明显失衡或真空，随后回撤到那个已知区间，等价格在区间内收住后，再去做下一段扩张
- ICT 后面还补了一句很重要：一旦理解了这三种特性，你会发现它们几乎对应所有市场环境，分别可以覆盖 `反转`、`趋势中的回撤`、以及 `震荡后走出的扩张`
- 这也解释了为什么这段非常值得重视：他不是在列举主观偏好的“招式”，而是在压缩成三类可迁移、可复用、跨时间框架都存在的底层结构

## 要点总结

- 这节课最重要的，不是机械区分月线周线日线，而是理解不同时间框架在模型里的分工
- 对日内交易者而言，更准确的说法不是“只看日线”，而是至少同时看 `日线 + 4H`
- 日线负责建立偏见与方向框架，4H 负责把高时间框架逻辑细化为更接近执行的结构上下文
- 对我而言，这节课最值得继续追踪的具体 setup，是 `下跌途中的海龟汤`，因为它天然把方向、流动性清扫与执行位置连到了一起
- 如果沿用自己的术语体系，那么 `下跌途中的海龟汤` 可以理解为看跌前提下的一种 `弹簧` 事件
- ICT 在实盘讲到的三种模式值得单独记住：`订单块回撤`、`海龟汤/扫止损`、`流动性真空回补后再扩张`
- 这三种模式更像底层市场条件，而不只是三个孤立 setup；它们很可能就是后续最值得研究的量化因子来源
- setup 不应越学越多，而应逐步收敛成少数自己真正看得懂、拿得住、做得出统计优势的类型

## 量化部分

- 对量化而言，这节课最有价值的地方在于：`日线不是执行层，而是更高一级的上下文层、过滤层、偏见层`
- 如果目标是日内交易，更合理的多层结构应该是：`日线定 bias -> 4H 定局部结构与关键区域 -> 更低时间框架做触发与执行`
- 也就是说，不要一上来就在低时间框架里机械找形态；更合理的顺序是先用日线定义市场状态，再用 4H 缩小到可执行区域，最后才看具体入场
- 因此，日线在量化里更适合承担 `regime filter`，4H 更适合承担 `setup filter` 或 `location refinement` 的角色
- 后续可重点研究三种框架的差异：`仅低时间框架信号`、`日线 + 低时间框架`、`日线 + 4H + 低时间框架`，这会比孤立比较某个形态更接近真正可落地的建模方向
- 如果进一步聚焦到我更关心的 `下跌途中的海龟汤`，那么一个更具体的研究方向就是把它与已有的 `OTE + 止损模式` 结合：先要求日线或 4H 保持下跌前提，再要求价格回到某个 OTE 区域附近，同时伴随对局部高低点止损的清扫，最后再看是否出现回收与顺势延续
- 这样做的意义在于，`OTE` 提供的是潜在定价区间，`止损模式 / 海龟汤` 提供的是流动性事件与触发确认；两者结合后，比单独交易 OTE 回撤或单独交易一次扫损更接近结构化 setup
- 如果采用你现在的命名，那么这里也可以把它进一步抽象成 `弹簧因子`：高时间框架给出前提，OTE 给出候选定价区，Sweep/海龟汤给出清扫事件，而回收与释放给出最终触发
- 如果把 54 分钟附近那三种模式翻译成量化语言，它们很可能可以视为三类核心因子：`OB 因子`、`Sweep 因子`、`Void/Imbalance 因子`
- 进一步说，这三类因子可以分别对应三种不同的市场交付机制：`回到机构定价区后继续交付`、`先清扫流动性再反向交付`、`失衡回补完成后恢复扩张`
- 它们之所以重要，是因为这已经不再是“看哪个图形更像”，而是在描述价格推进的三种底层路径；如果后续回测有效，就能围绕这三类条件构建统一框架，而不是为每个形态单独写一个策略
- 一个更贴近你当前理解的做法，是把这三种模式视为可组合因子而不是互斥策略：例如 `日线偏见 + 4H OTE 区域 + Sweep 因子`，或者 `高时间框架方向 + OB 因子 + Void 回补确认`
